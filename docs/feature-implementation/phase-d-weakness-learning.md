# 弱点分析 機能実装書

## 本文書について

本文書は、DEPクイズアプリにおける弱点分析と、それを学習行動へつなぐ機能群を説明する機能実装書である。学習履歴から弱点を判断し、対象を確認して復習し、結果を再分析へ戻す学習フィードバックループを扱う。

主な読者は、弱点分析や弱点学習機能を改修する前に、機能全体の意図と設計判断を把握したい開発者である。読後には、弱点分析が解決する課題、各機能が循環の中で担う役割、現在実装済みの範囲と将来候補の境界を説明できる。

本文書は全体像と設計意図を示すものであり、詳細な要件、変更差分、実装契約、保証内容は、それぞれIssue、PR、コード、テストなどの正本で確認する。

## 第1章 弱点分析の位置付け

問題を解いて学習履歴を保存できても、その履歴だけから次に何を学ぶべきかを判断することは難しい。弱点分析は、蓄積された履歴を次の学習判断へ変える入口である。

![学習履歴を次の学習判断に変える弱点分析の位置付け](images/phase-d-weakness-learning-section1.png)

図1が示すように、弱点分析は履歴を単なる過去の記録ではなく、重点的に見直す内容を考えるための判断材料に変える。目的は分析結果を表示することではなく、利用者が次の学習行動を選びやすくすることである。

ただし、判断を見るだけでは学習は完結しない。分析から対象確認、復習、履歴反映、再分析までがつながって初めて、履歴の変化を次の行動へ生かせる。第2章では、この学習フィードバックループの全体地図を示す。

## 第2章 弱点分析を起点とする学習フィードバックループ

分析結果を見るだけでは学習は進まない。弱点分析は終点ではなく、次の行動を決め、更新された学習履歴を再び判断へ戻す循環の入口である。

![弱点分析から対象確認、復習、進捗更新、再分析へつながる学習フィードバックループ](images/phase-d-weakness-learning-section2.png)

図2の循環では、分析を起点に具体的な対象を確認して復習し、その結果を既存の学習履歴へ戻す。これにより、復習の成果を含む最新の履歴から再分析でき、学習フィードバックループが閉じる。

学習履歴リセットはこの通常工程ではない。蓄積した評価履歴を前提とする判断を区切り、新しい学習周期へ移る必要がある場合の分岐として扱う。

以降の章では、循環の各部分を順に説明する。第3章は判断を作り、第4章は判断を行動へつなぎ、第5章は学習周期を切り替える。

## 第3章 学習履歴を学習判断へ変える ― 弱点分析の仕組み

学習履歴だけでは、次に何を見るべきか判断しづらい。弱点分析は、蓄積された履歴を次の学習判断へ変えるための入口である。

![学習履歴を学習判断へ変換する弱点分析の仕組み](images/phase-d-weakness-learning-section3.png)

図3では、学習履歴を複数の観点から整理し、判断材料へ変換する流れを示している。読み手がここで押さえるべき点は、低い正答率を機械的に同じ弱点として扱わないことである。未学習、データ不足、実際に誤答が残る弱点は学習上の意味が異なるため、区別して判断できることが重要になる。

また、弱点分析は固定された成績表ではない。復習や通常学習によって履歴が更新されれば、次に見るべき内容も変わるため、常に最新の学習履歴から現在の判断を導く。

ただし、判断材料が得られても、それだけでは復習行動に移れない。次章では、分析結果を具体的な対象確認と復習へつなぐ役割を説明する。

## 第4章 分析を学習行動へつなぐ ― 対象問題一覧と弱点起点の復習

分析結果を見るだけでは、学習行動にはならない。弱点として示された内容を、利用者が確認できる具体的な復習対象へつなぐ必要がある。

![分析結果を対象問題一覧と弱点起点の復習へつなぐ流れ](images/phase-d-weakness-learning-section4.png)

図4では、分析結果から対象問題一覧、弱点起点の復習、進捗反映へ進む流れを示している。この流れで特に押さえるべき点は、対象問題一覧を分析と復習の橋渡しとして置くことである。利用者は、弱点条件に対応する問題群を確認することで、分析結果を「実際に取り組む問題」として受け止められる。

復習結果は専用の一時履歴に閉じず、既存の学習履歴へ戻す。これにより、弱点起点で解いた結果が次回以降の分析材料に含まれ、復習後の状態を再分析できる。

一方で、履歴を更新し続けるだけでなく、試験前後や学習方針の変更に合わせて一区切りにしたい場面もある。次章では、新しい学習周期へ移るための学習履歴リセットを説明する。

## 第5章 学習周期を切り替える ― 学習履歴リセット

学習履歴リセットは、すべてを消す操作ではなく、これまでの評価履歴を一区切りにして新しい学習周期へ移るための手段である。

![学習履歴を一区切りにし学習資産を残して新しい周期へ移るリセットの考え方](images/phase-d-weakness-learning-section5.png)

図5では、評価履歴を区切りながら、学習資産を残して次の周期へ移る考え方を示している。ここで重要なのは、評価履歴と学習資産を同じものとして扱わないことである。学習結果や誤答理由は現在の状態を判断する材料であり、新しい学習周期へ切り替えるときには一区切りにする。

一方で、利用者が育てたメモ、ブックマーク、設定のような学習資産は、新しい学習周期でも足場として再利用する。履歴は区切り、資産は再利用する。この考え方は、弱点分析、復習、リセットを安全に循環させるための設計思想につながる。次章では、その主要な設計思想を整理する。

## 第6章 弱点分析を支える主要な設計思想

弱点分析を学習フィードバックループとして成立させるには、分析・対象確認・復習・学習履歴リセットを、同じ状態管理の考え方でつなぐ必要がある。本章では、その循環を破綻させないための設計思想を整理する。

![弱点分析を支える4つの主要な設計思想](images/phase-d-weakness-learning-section6.png)

図6の各思想は、独立したルールではなく、状態重複、分析の陳腐化、体験の分断、意図しない更新を同時に避けるために組み合わされる。これにより、分析結果の整合性と鮮度を保ちながら、既存の学習体験を再利用し、状態変更を利用者の明示的な操作に限定できる。

この設計は、弱点を確認してから復習し、その結果を最新の学習履歴として再分析へ戻す循環の信頼性を支える。第7章では、この思想の上で現在どこまで実現済みかと、まだ現在の機能として扱わない将来候補を分けて説明する。

## 第7章 現在地と今後の発展方向

弱点分析は、最新の学習履歴から現在の弱点を見つけ、対象確認と復習へつなげる段階までを現在実装済みとしている。一方で、今後考えられる拡張を、現在実装済みの機能と同じ確度では扱わない。

![弱点分析の現在地・現在の境界・今後の発展方向](images/phase-d-weakness-learning-section7.png)

図7の将来候補は、保証済み機能や確定ロードマップではない。実現するには、どの履歴を母数にするか、どの評価指標で効果を測るか、どの判断や推薦を説明可能にするかといった追加設計が必要になる。

したがって、本文書では現在実装済みの機能を、最新の学習履歴にもとづく分析、対象確認、復習、再分析の学習フィードバックループとして位置付ける。将来候補は、その判断材料をさらに活用する方向として扱い、現在の境界と明確に分離する。第8章では、現在地や境界を詳しく確認するための正本を案内する。
