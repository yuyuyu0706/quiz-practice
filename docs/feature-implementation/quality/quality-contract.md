# 機能実装書の品質契約

## 関連文書

- 品質ルールと人手レビュー境界：本書
- 実行・障害対応・追加変更手順：[運用ガイド](operation-guide.md)

## 目的と適用範囲

本書は、`docs/feature-implementation/*.md` に置く機能実装書について、機械的に検証する品質と人手レビューで確認する品質の境界を定義する正本である。初期適用対象は [Phase D 弱点分析の機能実装書](../phase-d-weakness-learning.md) とする。本書自身は機能実装書ではないため、validator の対象外である。

対象は Markdown の構造、ローカル参照、未解決の定型記述、および文書ごとに確定した構成である。対象外は validator 本体、npm script、CI、外部 URL の到達確認、アンカーの妥当性確認、インライン抑制、および自然言語処理による文章採点である。

## 品質契約の原則

1. 自動検証は、同じ入力に対して常に同じ結果を返す。
2. CI を止めるのは、修正方法が明確な違反に限定する。
3. 意味、事実の正確性、読みやすさ、設計判断を単純なキーワード一致で判定しない。
4. 全機能実装書に適用する共通ルールと、文書固有ルールを分離する。
5. 誤検知を個別除外で回避せず、検出条件、除外条件、または品質契約を修正する。
6. ルールを追加・変更するときは、回帰テストも追加・更新する。
7. 自動検証の通過は、内容の正確性や設計妥当性を保証しない。

## 識別子、重要度、共通の解釈

| 区分                     | Rule ID      |
| ------------------------ | ------------ |
| 全機能実装書の共通ルール | `FDOC-Cxxx`  |
| Phase D 文書固有ルール   | `FDOC-PDxxx` |
| 人手レビュー項目         | `FDOC-Mxxx`  |

| 重要度  | 意味と終了コード                                                                                   |
| ------- | -------------------------------------------------------------------------------------------------- |
| ERROR   | 明確かつ再現可能で、修正方法が明確な違反。1件以上で exit code 1。                                  |
| WARNING | 機械検出できるが、修正が常に必要ではない指摘。WARNING のみなら exit code 0。初期版では定義しない。 |
| MANUAL  | 意味・正確性・可読性など人の判断が必要な観点。validator の検出対象外。                             |

特記しない限り、コードブロック内の Markdown 風記述は検出対象外とする。インラインコードも `FDOC-C007` と `FDOC-C008` の対象外とする。外部 URL は `http://`、`https://` の URL とし、存在・到達性を確認しない。

## 共通自動検証ルール

### FDOC-C001：文書が空でない

| 項目             | 定義                                                     |
| ---------------- | -------------------------------------------------------- |
| 適用範囲・重要度 | 共通・ERROR                                              |
| 検出条件         | 空文字、または空白文字だけで構成される文書を違反とする。 |
| 除外条件         | なし。                                                   |
| メッセージ       | `文書本文がありません`                                   |
| 正常例           | `# 題名`                                                 |
| 異常例           | 空ファイル、または `   ` のみのファイル。                |
| テスト観点       | 空、空白のみ、本文ありを fixture で確認する。            |

### FDOC-C002：H1 が1つだけ存在する

| 項目             | 定義                                                     |
| ---------------- | -------------------------------------------------------- |
| 適用範囲・重要度 | 共通・ERROR                                              |
| 検出条件         | ATX 形式の H1 が0件または2件以上である。                 |
| 除外条件         | コードブロック内の `#` は見出しに数えない。              |
| メッセージ       | `H1 は1つだけにしてください`                             |
| 正常例           | `# 題名` が1つだけある。                                 |
| 異常例           | H1 がない、または `# 題名` が2つある。                   |
| テスト観点       | 0件、1件、複数件、コードブロック内の疑似 H1 を確認する。 |

### FDOC-C003：見出しレベルが飛ばない

| 項目             | 定義                                                                                        |
| ---------------- | ------------------------------------------------------------------------------------------- |
| 適用範囲・重要度 | 共通・ERROR                                                                                 |
| 検出条件         | 直前の見出しから2階層以上深い見出しへ進む（例: H2 から H4）。最初の見出しは H1 として扱う。 |
| 除外条件         | 見出しを浅く戻すことと、コードブロック内は違反にしない。                                    |
| メッセージ       | `見出しレベルが1階層を超えて深くなっています`                                               |
| 正常例           | `# 題名`、`## 章`、`### 節`。                                                               |
| 異常例           | `# 題名` の直後に `### 節`。                                                                |
| テスト観点       | H1→H3、H2→H4、深さを戻す場合、コードブロックを確認する。                                    |

### FDOC-C004：同じ親の配下に重複見出しがない

| 項目             | 定義                                                                                     |
| ---------------- | ---------------------------------------------------------------------------------------- |
| 適用範囲・重要度 | 共通・ERROR                                                                              |
| 検出条件         | 同一親・同一階層に完全一致する見出しが複数ある。親は直近の1階層浅い見出しとする。        |
| 除外条件         | 異なる親の配下、または異なる階層で同じ語を使うことは許可する。コードブロック内は対象外。 |
| メッセージ       | `同じ親の配下に重複する見出しがあります`                                                 |
| 正常例           | 別々の H2 配下にそれぞれ `### 詳細` がある。                                             |
| 異常例           | 同じ H2 配下に `### 詳細` が2つある。                                                    |
| テスト観点       | 同一親の重複、別親での同名、階層違い、完全一致判定を確認する。                           |

### FDOC-C005：ローカル画像の参照先が存在する

| 項目             | 定義                                                                                          |
| ---------------- | --------------------------------------------------------------------------------------------- |
| 適用範囲・重要度 | 共通・ERROR                                                                                   |
| 検出条件         | Markdown 画像記法の相対パスを当該 Markdown ファイル基準で解決し、参照先ファイルが存在しない。 |
| 除外条件         | 外部画像 URL は到達確認しない。コードブロック内の画像記法は対象外。                           |
| メッセージ       | `参照画像が存在しません`                                                                      |
| 正常例           | `![図](images/diagram.png)` のファイルが存在する。                                            |
| 異常例           | `![図](images/missing.png)` のファイルが存在しない。                                          |
| テスト観点       | 相対パス、親ディレクトリ参照、存在しない参照、外部 URL、コードブロックを確認する。            |

### FDOC-C006：ローカル文書リンクの参照先が存在する

| 項目             | 定義                                                                                                                                                              |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 適用範囲・重要度 | 共通・ERROR                                                                                                                                                       |
| 検出条件         | Markdown リンクの相対ファイルパスを当該 Markdown ファイル基準で解決し、参照先が存在しない。フラグメントは解決前に除く。                                           |
| 除外条件         | 外部 URL とアンカーの妥当性は確認しない。`[第3章へ](#第3章)` のような文書内アンカーのみのリンクは検証対象外とする。画像リンクとコードブロック内のリンクは対象外。 |
| メッセージ       | `参照文書が存在しません`                                                                                                                                          |
| 正常例           | `[詳細](guide.md#usage)` の `guide.md` が存在する。                                                                                                               |
| 異常例           | `[詳細](missing.md)` のファイルが存在しない。                                                                                                                     |
| テスト観点       | 相対パス、フラグメント付きリンク、アンカーのみのリンク、存在しない参照、外部 URL、画像記法を確認する。                                                            |

### FDOC-C007：ローカル環境依存パスを含まない

| 項目             | 定義                                                                                                                                                                    |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 適用範囲・重要度 | 共通・ERROR                                                                                                                                                             |
| 検出条件         | URL 部分を除く本文に、一般的な Windows ユーザーディレクトリを表す任意のドライブレターの正規表現 `[A-Za-z]:\\Users\\`、`/Users/`、`/home/`、`file://` のいずれかを含む。 |
| 除外条件         | `http://` または `https://` の URL 部分、コードブロック、インラインコードは対象外。リポジトリ内の相対パスは許可する。                                                   |
| メッセージ       | `ローカル環境依存パスを含めないでください`                                                                                                                              |
| 正常例           | `docs/guide.md`、`https://example.com/home/guide`。                                                                                                                     |
| 異常例           | `C:\\Users\\example\\project`、`D:\\Users\\example\\project`、`/Users/example/project`。                                                                                |
| テスト観点       | `C:` 以外を含む各ドライブレターの Windows パス、Unix 系パス、`file://`、相対パス、URL 内の `/home/`、コードブロック、インラインコードを確認する。                       |

### FDOC-C008：未解決プレースホルダーを含まない

| 項目             | 定義                                                                                                        |
| ---------------- | ----------------------------------------------------------------------------------------------------------- |
| 適用範囲・重要度 | 共通・ERROR                                                                                                 |
| 検出条件         | 本文に `TODO`、`TBD`、`FIXME`、`{{PLACEHOLDER}}`、`<未入力>` のいずれかを含む。英字語は単語境界で判定する。 |
| 除外条件         | コードブロックとインラインコードは対象外。列挙した定型語以外の HTML・説明用記法は対象外。                   |
| メッセージ       | `未解決プレースホルダーを解消してください`                                                                  |
| 正常例           | `今後の検討事項を記録する。`                                                                                |
| 異常例           | `TODO: 詳細を追記する。`                                                                                    |
| テスト観点       | 各定型語、英字語の部分一致を除外すること、コードブロック、インラインコードを確認する。                      |

## Phase D 文書固有の自動検証ルール

対象は `docs/feature-implementation/phase-d-weakness-learning.md` だけである。

### FDOC-PD001：H1 が確定名称と一致する

| 項目             | 定義                                                  |
| ---------------- | ----------------------------------------------------- |
| 適用範囲・重要度 | 文書固有・ERROR                                       |
| 検出条件         | 唯一の H1 が `弱点分析 機能実装書` と完全一致しない。 |
| 除外条件         | H1 以外の本文は対象外。                               |
| メッセージ       | `H1 は「弱点分析 機能実装書」にしてください`          |
| 正常例           | `# 弱点分析 機能実装書`                               |
| 異常例           | `# Phase D 弱点分析`                                  |
| テスト観点       | 一致、不一致、前後の空白、H1 不在を確認する。         |

### FDOC-PD002：必須章がすべて存在する

| 項目             | 定義                                                                                   |
| ---------------- | -------------------------------------------------------------------------------------- |
| 適用範囲・重要度 | 文書固有・ERROR                                                                        |
| 検出条件         | 次の H2 のいずれかが完全一致で存在しない。                                             |
| 除外条件         | `目次` は導入部として許可し、必須章一覧には含めない。H3 以下とコードブロックは対象外。 |
| メッセージ       | `必須章がありません`                                                                   |
| 正常例           | 下表の導入部と8章がすべて H2 として存在する。                                          |
| 異常例           | `第5章 学習周期を切り替える ― 学習履歴リセット` がない。                               |
| テスト観点       | 各章の欠落、H3 だけの同名見出し、表記変更を確認する。                                  |

| 順序 | 必須 H2                                                     |
| ---- | ----------------------------------------------------------- |
| 1    | 本文書について                                              |
| 2    | 第1章 弱点分析の位置付け                                    |
| 3    | 第2章 弱点分析を起点とする学習フィードバックループ          |
| 4    | 第3章 学習履歴を学習判断へ変える ― 弱点分析の仕組み         |
| 5    | 第4章 分析を学習行動へつなぐ ― 対象問題一覧と弱点起点の復習 |
| 6    | 第5章 学習周期を切り替える ― 学習履歴リセット               |
| 7    | 第6章 弱点分析を支える主要な設計思想                        |
| 8    | 第7章 現在地と今後の発展方向                                |
| 9    | 第8章 参考情報                                              |

### FDOC-PD003：章の順序が正しい

| 項目             | 定義                                                                     |
| ---------------- | ------------------------------------------------------------------------ |
| 適用範囲・重要度 | 文書固有・ERROR                                                          |
| 検出条件         | FDOC-PD002 の表にある H2 が、表の順序どおりに出現しない。                |
| 除外条件         | 必須章以外の H2 は許可する。コードブロック内は対象外。                   |
| メッセージ       | `必須章の順序が正しくありません`                                         |
| 正常例           | `本文書について` の後に第1章から第8章が並ぶ。                            |
| 異常例           | 第4章が第3章より前にある。                                               |
| テスト観点       | 隣接章の入替、離れた章の入替、追加 H2 があっても通過することを確認する。 |

### FDOC-PD004：図1〜8の参照が存在する

| 項目             | 定義                                                                                                       |
| ---------------- | ---------------------------------------------------------------------------------------------------------- |
| 適用範囲・重要度 | 文書固有・ERROR                                                                                            |
| 検出条件         | `images/phase-d-weakness-learning-section1.png` から `section8.png` のいずれかが画像参照として存在しない。 |
| 除外条件         | 代替テキストと画像の意味内容は対象外。コードブロック内は対象外。                                           |
| メッセージ       | `必須の章図参照がありません`                                                                               |
| 正常例           | 8つすべてを `![代替テキスト](...)` で参照する。                                                            |
| 異常例           | `section6.png` の参照がない。                                                                              |
| テスト観点       | 各参照の欠落、単なる本文文字列ではなく画像記法であることを確認する。                                       |

### FDOC-PD005：各図が対応する章内に配置されている

| 項目             | 定義                                                                                |
| ---------------- | ----------------------------------------------------------------------------------- |
| 適用範囲・重要度 | 文書固有・ERROR                                                                     |
| 検出条件         | `sectionN.png` が第N章の H2 から次の H2 の直前までにない。                          |
| 除外条件         | 導入部と第8章後の本文は対象外。図の意味内容、代替テキスト、サイズ指定は判定しない。 |
| メッセージ       | `章図が対応する章内にありません`                                                    |
| 正常例           | 第3章の範囲で `section3.png` を参照する。                                           |
| 異常例           | 第2章の範囲で `section3.png` を参照する。                                           |
| テスト観点       | 各章の正配置、別章への配置、章外への配置を確認する。                                |

### FDOC-PD006：読者向け見出しに内部管理名を使わない

| 項目             | 定義                                                                                 |
| ---------------- | ------------------------------------------------------------------------------------ |
| 適用範囲・重要度 | 文書固有・ERROR                                                                      |
| 検出条件         | H1 または H2 に `Phase D`、`Lv3`、`Lv4`、`DX-`、`Issue #`、`PR #` のいずれかを含む。 |
| 除外条件         | 本文、H3 以下、第8章の参照ナビゲーション、コードブロックは対象外。                   |
| メッセージ       | `読者向け見出しに内部管理名を使用しないでください`                                   |
| 正常例           | `## 第1章 弱点分析の位置付け`                                                        |
| 異常例           | `## Phase D の設計思想`                                                              |
| テスト観点       | 禁止語ごとに H1/H2 を確認し、本文と H3 では通過することを確認する。                  |

## 人手レビュー項目

| Rule ID   | 確認観点                                                                                     |
| --------- | -------------------------------------------------------------------------------------------- |
| FDOC-M001 | 現行実装との事実整合。未実装・廃止済みの挙動を実装済みと記述していないか。                   |
| FDOC-M002 | 章間の因果関係。課題→学習循環→各機能→設計思想→現在地→詳細参照が自然につながるか。            |
| FDOC-M003 | 図と本文の責務分担。図の逐語的な繰返しを避け、本文が理由・判断・境界を補っているか。         |
| FDOC-M004 | 現在実装済みと将来候補の境界。将来候補を保証済み機能や確定ロードマップとして扱っていないか。 |
| FDOC-M005 | キーワードと強調表現。章の主要メッセージが把握でき、強調が過剰でないか。                     |
| FDOC-M006 | 正本ナビゲーション。Issue、PR、コード、テストの役割と確認先が正しいか。                      |
| FDOC-M007 | 読者にとっての可読性。内部用語への依存、長すぎる文、重複、前提飛躍がないか。                 |

これらはすべて MANUAL であり、意味判断をキーワード検索へ置き換えない。レビューでは、各項目に対し現行コード・テスト・関連成果物を照合して判断する。

## 例外・抑制方針

初期版では `<!-- validator-disable FDOC-C008 -->` のような文書内インライン抑制を導入しない。例外が必要になったときは、共通ルールの検出条件・除外条件を修正するか、文書固有ルールとして差異を明示する。例外またはルール変更には、後続工程で対応する回帰 fixture を必ず追加する。

## 後続 validator・回帰テストへの引継ぎ

ERROR の診断は少なくとも対象ファイル、Rule ID、行番号、問題内容、修正対象を表示する。違反箇所を特定できる場合は実際の行番号を表示し、必須要素の欠落や文書全体に対する違反で行番号を特定できない場合は `Line: -` とする。修正対象を特定できない場合は `Target: -` とする。出力形式は次を固定する。

```text
[ERROR] docs/feature-implementation/example.md
Rule: FDOC-C005
Line: 42
Message: 参照画像が存在しません
Target: images/example-section3.png
```

行番号または Target を特定できない代表例は次のとおりである。

```text
[ERROR] docs/feature-implementation/example.md
Rule: FDOC-C002
Line: -
Message: H1 は1つだけにしてください
Target: -

[ERROR] docs/feature-implementation/phase-d-weakness-learning.md
Rule: FDOC-PD002
Line: -
Message: 必須章がありません
Target: 第3章 学習履歴を学習判断へ変える ― 弱点分析の仕組み

[ERROR] docs/feature-implementation/phase-d-weakness-learning.md
Rule: FDOC-PD004
Line: -
Message: 必須の章図参照がありません
Target: images/phase-d-weakness-learning-section3.png
```

最初の例は H1 欠落、2つ目は必須章欠落、3つ目は必須画像参照欠落を表す。欠落した要素そのものを修正対象として特定できる場合は `Target` に表示し、特定できない場合だけ `Target: -` を表示する。
